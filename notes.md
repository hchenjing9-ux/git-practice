# Git 学习记录

## 基本概念

- repository：保存项目文件和 Git 历史的仓库
- add：把工作区修改加入暂存区
- commit：把暂存内容保存为本地历史快照
- branch：从某个提交创建独立的开发路线
- push：把本地提交上传到远程仓库
- pull：获取并整合远程仓库的更新
- merge：合并不同分支的历史
- conflict：Git 无法自动决定如何合并时产生的冲突

## 遇到的问题

首次 push 时提示远程仓库包含本地没有的提交。

## 解决过程

先使用 fetch 获取远程历史，然后使用 merge 合并。本地和远程都创建了 README，因此产生冲突。手动选择最终内容、使用 add 标记冲突已解决，再完成合并提交。
